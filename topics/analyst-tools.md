# Инструменты системного аналитика

Системный аналитик работает на стыке бизнеса, разработки, тестирования и эксплуатации. Поэтому набор инструментов должен покрывать не только сбор требований, но и проверку API, анализ данных, работу с документацией и диагностику проблем.

## Анализ и исследование

| Инструмент | Назначение | Комментарий |
| :--- | :--- | :--- |
| Google | Поиск доменной и технической информации | Базовый вход для исследований и факт-чекинга |
| ChatGPT / LLM | Черновики требований, разбор материалов, генерация вопросов | Использовать с обязательной верификацией |
| Miro | Discovery-сессии и совместный анализ | Удобен для workshop-формата |

## Документация и знания

| Инструмент | Назначение | Комментарий |
| :--- | :--- | :--- |
| Confluence | База знаний и требования | Стандарт в корпоративной среде |
| Notion | Личная/командная база знаний | Быстрый старт для небольших команд |
| Markdown + Git | Версионирование критичных документов | Основа практик DocOps |

## API и интеграции

| Инструмент | Назначение | Комментарий |
| :--- | :--- | :--- |
| Postman | Тестирование API и коллекции запросов | Стандарт де-факто для API-проверок |

| Browser DevTools | Анализ сетевых вызовов в браузере | Быстрое воспроизведение дефектов |

| dbdiagram.io | Визуализация схем БД | Удобно для быстрых ER-моделей |

## DevOps и эксплуатация

| Инструмент | Назначение | Комментарий |
| :--- | :--- | :--- |
| Git | Версионирование артефактов аналитики | Ветки, review, трассировка изменений |
| CI/CD для документации | Автопроверка и публикация docs | Практика DocOps |
| Мониторинг (Prometheus, Grafana, ELK) | Анализ инцидентов и метрик | Важен для анализа production-проблем |

## Что изучать в первую очередь

1. Jira + Confluence (операционная база аналитика).
2. Git + Markdown (управление изменениями и DocOps-подход).
3. Postman + DevTools (проверка API и быстрый теханализ).
4. DBeaver (валидация гипотез на данных).
5. LLM-инструменты (ускорение рутинных задач).
